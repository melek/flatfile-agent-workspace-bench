# Observations

Append-only log of descriptive notes — norms, patterns, framings, half-formed thoughts that aren't decisions.

**The bar is low.** This file is *descriptive*, not *prescriptive*: "this is how it tends to go" rather than "we decided X." Use it when you notice something worth recognizing again later but it doesn't constrain future action.

**The highest-value entries are cross-cutting** — patterns visible across multiple sessions, projects, or domains. A single-session observation usually belongs in today's daybook Log; promote to here when you see the same shape again.

**When uncertain, default to today's daybook Log** and promote later if the pattern earns it.

**Without `observations.md`:** cross-cutting patterns get noticed once, lost in the daybook chronology, then re-noticed weeks later as if new. The cost compounds — you re-recognize "credential provisioning is always the bottleneck" three separate times before doing anything about it. Patterns that should crystallize into decisions never do because there's no record they keep happening.

## Template

```markdown
## YYYY-MM-DD — Short title

**Generated-by:** inference | user | mixed

Description of the observation. One paragraph or a few sentences. Norms, patterns, framings, half-formed thoughts. Doesn't have to be a decision.
```

The **Generated-by** field marks who produced this entry: the agent (`inference`), the user (`user`), or both (`mixed`). See `methodology.md#provenance-markers` — the word is "inference," not "drafted," because token production is calculation, not authorship.

## Shape

An observation entry is short and descriptive:

> ## YYYY-MM-DD — Short title naming the pattern
>
> **Generated-by:** inference | user | mixed
>
> One paragraph describing the pattern. Specific instances if you have them. *Descriptive, not prescriptive* — doesn't bind future action.

If an observation *leads* to a decision, write the decision in `decisions.md` and reference this observation in its Context. The two files reinforce each other when patterns crystallize into commitments.
